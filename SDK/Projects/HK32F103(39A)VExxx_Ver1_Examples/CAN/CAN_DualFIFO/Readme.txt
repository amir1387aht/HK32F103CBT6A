******************************例程详细描述*****************************
功能描述:
	本例程描述了如何配置CAN使得多个board之间能够互相发送和接收数据帧。
	通过调用库函数CAN_Init()和CAN_FilterInit()来完成对CAN的配置，两个board相互通信的方法如下：	
	1.CAN的波特率配置为500Kbit/s
	2.使能CAN的CAN_IT_FMP0和CAN_IT_FMP1中断
	3.将程序分别下载到两个评估板A和B，将A板的CAN和B板的CAN通过CAN总线连接
	4.每次按下A或B板按IKEY1，都会使得发送的DATA-1，其余不变，示例：
	若第一次按下按键KEY1，则会发送一帧标准帧为：ID:0x321 DLC:1 DATA:0x04，再按下一次按键KEY1，
	则发送一帧标准帧为：ID:0x321 DLC:1 DATA:0x03，若DATA<0x02则，重新从0x05递减
	5.按下A或B板按键KEY2，都会使得发送的DATA+1，其余不变，示例：
	若第一次按下按键KEY2，则发送一帧标准帧为：ID:0x123 DLC:1 DATA:0x01，再按下一次按键KEY2，
	则发送一帧标准帧为：ID:0x123 DLC:1 DATA:0x02，若DATA=0x04则，重新从0x00递增
	6.在中断函数中CAN的FIFO0和FIFO1中断中接收发送帧，其中FIFO0通过过滤器配置只接收帧ID为0x321的帧，FIFO1通过过滤器配置只接收帧ID为0x123的帧
	7.接收和发送帧的DATA为0x01、0x02、0x03和0x04时，board上的LED会表现出LED1亮、LED2亮、LED3亮和LED1、LED2、LED3全亮

时钟配置:
	本例程采用HSE+PLL72MHz作为系统时钟，用户如果需要更改系统时钟配置，请参考对应用户手册或者
	RCC库函数及RCC相关例程。

适用芯片:
	HK32F103xC,HK32F103xD,HK32F103xE,HK32F103xxxxA全系列

涉及外设:
	CAN/GPIO/NVIC/RCC
	
涉及管脚:	
	PA11 ---------> CAN RX
	PA12 ---------> CAN TX
	PA0 ---------> KEY1
	PA1 ---------> KEY2
	PB1---------> LED1 
	PB0---------> LED2 
	PB5 ---------> LED3 

注意事项:
	1. 推荐使用MDK5.00及其以上版本编译本工程（MDK-ARM）
	          或者IAR8.22及其以上版本编译本工程(EWARM);
	2. 使用之前检测电源VCC、GND等是否正常。

如何使用:
	1. 直接编译程序下载到评估版A和B，重启评估版;
	2. 将A板的CAN和B板的CAN以CAN总线的方式连接
	3. 按下A板按键KEY1或KEY2，观察LED亮灭情况判断是否A板发送，同时观察B板LED亮灭情况判断是否正常接收；
