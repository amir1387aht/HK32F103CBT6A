******************************例程详细描述*****************************
功能描述:
	本例程展示了如何定期更新IWDG重装载计数器，以及如何模拟软件故障，在编程时间到期时没有喂狗，从而生成MCU IWDG复位。
	1. IWDG超时时间设置为250ms；
	2. LSI在内部连接TIM5的通道4，配置TIM去测量LSI的频率，目的是校准IWDG的时钟；
	3. IWDG重装载计数器在每240ms进行刷新避免IWDG产生复位；
	4. 串口在每240ms进行打印“程序正在运行”，表明程序正在运行；
	5. 按下KEY1按键，外部EXTI上升沿中断模拟软件故障，一旦EXTI line事件发生，会在此中断中操作一个非法地址，造成程序进入hardfault，
	    进而导致程序不能正常的喂狗，最终导致IWDG复位，串口打印“复位”。
                	
时钟配置:
	本例程采用HSE72MHz作为系统时钟，用户如果需要更改系统时钟配置, 请参考对应用户手册或者
	RCC库函数及RCC相关例程。

适用芯片:
	HK32F103xC,HK32F103xD,HK32F103xE,HK32F103xxxxA全系列

涉及外设:
	IWDG,USART1,GPIO,TIM
	
涉及管脚:	
	USART1_TX(PA9),    USART1_RX(PA10),     KEY1(PA0)

注意事项:
	1. 推荐使用MDK5.00及其以上版本编译本工程（MDK-ARM）
	    或者IAR8.22及其以上版本编译本工程(EWARM);
	2. 使用之前检测电源VCC、GND等是否正常。

如何使用:
	1. 直接编译程序下载到评估版，重启评估版;
	2. 连接串口，串口打印“程序正在运行”，表明程序在定时喂狗；
	3. 按下KEY1按键，串口打印“复位”。
